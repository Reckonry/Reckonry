# Roadmap

This roadmap describes intended direction, not a guarantee of delivery dates. Reckonry prioritizes evidence-based financial records, privacy, and reviewable tax-adjacent outputs.

## v0.1.0-alpha

- Done: Stabilize the canonical ledger v1 foundation.
- Done: Provide Binance importer coverage for the synthetic public demo workflow and unsupported row preservation.
- Done: Produce Italy RW accountant outputs from ledger evidence.
- Done: Generate an initial Tax Dossier for professional review.
- Done: Build audit and integrity reports with source summaries and reproducible artifacts.
- Done: Document current CLI workflows and limitations.

## v0.2.0

- Planned: Improve importer diagnostics and mapping coverage.
- Planned: Add more importer fixtures using fake or anonymized sample files only.
- Planned: Extend audit package validation and reconciliation reporting.
- Planned: Refine Tax Dossier structure for accountant handoff.
- Planned: Formalize Plugin SDK contracts for importers and reports.

## v0.3.0

- Planned: Add more importers behind documented importer contracts.
- Planned: Expand reconciliation beyond the initial Binance flow.
- Planned: Introduce early external plugin loading.
- Planned: Strengthen ledger schema compatibility checks and migration guidance.
- Planned: Publish clearer SDK examples for importer, report, reconciliation, and tax extensions.

## v1.0.0

- Planned: Declare v1 canonical ledger schema compatibility expectations.
- Planned: Provide documented CLI compatibility policy.
- Planned: Maintain documented importer compatibility rules.
- Planned: Provide documented audit package output expectations for professional review workflows.
- Planned: Support a broader importer ecosystem through the Plugin SDK.
- Planned: Keep country-specific tax modules isolated from Core and backed by official sources.
