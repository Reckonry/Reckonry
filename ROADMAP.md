# Roadmap

This roadmap describes intended direction, not a guarantee of delivery dates. Reckonry prioritizes verifiable financial evidence, privacy, and reviewable tax logic.

## v0.1.0-alpha

- Stabilize the canonical ledger v1 foundation.
- Expand Binance importer coverage for common exports and unsupported row preservation.
- Produce Italy RW accountant outputs from ledger evidence.
- Generate an initial Tax Dossier for professional review.
- Build an audit package with integrity checks, source summaries, and reproducible artifacts.
- Document current CLI workflows and limitations.

## v0.2.0

- Improve importer diagnostics and mapping coverage.
- Add more importer fixtures using fake or anonymized sample files only.
- Extend audit package validation and reconciliation reporting.
- Refine Tax Dossier structure for accountant handoff.
- Formalize Plugin SDK contracts for importers and reports.

## v0.3.0

- Add more importers behind stable importer contracts.
- Expand reconciliation beyond the initial Binance flow.
- Introduce early external plugin loading.
- Strengthen ledger schema compatibility checks and migration guidance.
- Publish clearer SDK examples for importer, report, reconciliation, and tax extensions.

## v1.0.0

- Declare stable canonical ledger schema compatibility guarantees.
- Provide production-grade CLI compatibility policy.
- Maintain mature importer compatibility rules.
- Provide stable audit package outputs for professional review workflows.
- Support a broader importer ecosystem through the Plugin SDK.
- Keep country-specific tax modules isolated from Core and backed by official sources.
