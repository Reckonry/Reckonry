# Reckonry v0.2.0-alpha

Release date: 2026-07-04  
Tag: `v0.2.0-alpha`

## What Reckonry Is

Reckonry is open-source infrastructure for reconstructing, validating, auditing,
reconciling, and reviewing financial ledger evidence.

The project is not a tax calculator. It does not provide legal, tax,
accounting, financial, or investment advice. It generates reproducible and
explainable artifacts that can support professional review.

## What Changed Since v0.1.0-alpha

This alpha proves that Reckonry can extend beyond the initial Binance Italy
workflow without changing the canonical ledger model.

### Coinbase Importer

- Added a Coinbase CSV importer.
- Supports the synthetic public demo data and selected CSV fixture schemas.
- Generates canonical ledger events.
- Preserves unsupported rows as unknown ledger events.
- Participates in plugin discovery as an implemented alpha importer.

### Coinbase Reconciliation

- Added `Reckonry.Reconciliation.Coinbase`.
- Adds `coinbase-global` provider-level reconciliation.
- Compares synthetic aggregate statement metadata with generated Reckonry ledger
  counts.
- Writes `reconciliation-summary.json` and `reconciliation-summary.md`.

This reconciliation module is intentionally narrow. It does not perform balance
verification, valuation checks, tax reconciliation, official Coinbase document
extraction, or filing review.

### Coinbase Synthetic Demo Lane

The public demo now includes a second provider lane:

1. Import synthetic Coinbase CSV data.
2. Generate a Coinbase `ledger.json`.
3. Validate the generated ledger.
4. Generate ledger integrity output.
5. Reconcile against synthetic Coinbase aggregate statement metadata.

The Binance Italy demo remains the complete provider/country professional-review
workflow for this alpha.

### Plugin Templates

Added source-based `dotnet new` templates:

- `reckonry-importer`
- `reckonry-tax-module`
- `reckonry-report`
- `reckonry-reconciliation`

Each template includes source code, tests, fake sample data, README material, and
architecture notes. These templates are alpha scaffolding for external module
development. They are not a stable binary plugin SDK and are not published as
NuGet packages in this release.

### CLI Doctor

Added `reckonry doctor` diagnostics:

- `reckonry doctor`
- `reckonry doctor plugins`
- `reckonry doctor privacy`
- `reckonry doctor environment`
- `reckonry doctor sdk`
- `reckonry doctor demo`
- `reckonry doctor repository`

The command checks local environment, installed modules, ignored private paths,
sample data, SDK templates, demo health, and repository shape.

### CLI Explain

Added `reckonry explain` for developer/auditor provenance review.

The command explains supported structured artifacts step by step and points back
to source rows, ledger events, postings, and report rows where available.

Supported in this alpha:

- Canonical `ledger.json`.
- Ledger integrity JSON.
- RW snapshot JSON/CSV.
- RW value JSON/CSV.

PDF explanation is intentionally not supported because PDF structure is not
machine-verifiable enough for this command. Use generated structured companion
artifacts instead.

### Snapshot And Golden Tests

Added snapshot/golden tests for public user-facing artifacts:

- CLI output.
- Demo output.
- Audit Markdown.
- Reconciliation Markdown.
- Accountant package Markdown.
- RW CSV.
- RW JSON.
- Tax Dossier PDF structure.
- README command examples.

These tests use synthetic demo data only and are intended to catch accidental
wording, structure, formatting, and generated-artifact changes.

### Documentation Updates

Updated README, architecture, importer SDK, reconciliation SDK, and SDK template
documentation to reflect the current alpha implementation.

## Supported Demo Workflow

Run:

```bash
dotnet build Reckonry.sln
dotnet test Reckonry.sln
scripts/demo.sh
```

On Windows PowerShell:

```powershell
dotnet build Reckonry.sln
dotnet test Reckonry.sln
./scripts/demo.ps1
```

Generated demo outputs are written to ignored local files under
`artifacts/demo/`.

The demo still intentionally includes professional-review gaps and ends the
Italy RW/Tax Dossier path with `NOT READY FOR FILING`. That is expected.
Reckonry should expose missing evidence, not invent it.

## What This Alpha Does Not Include

- Tax filing.
- Tax certainty.
- Legal, tax, accounting, financial, or investment advice.
- Production API support.
- Hosted SaaS workflows.
- Authentication or database persistence.
- NuGet package publishing.
- Stable external SDK guarantees.
- Stable plugin binary compatibility guarantees.
- Complete Binance product coverage.
- Complete Coinbase product coverage.
- Real Kraken, Revolut, Crypto.com, or Bitstamp importers.
- Multi-country tax coverage beyond the Italy alpha module.
- Automatic market pricing.
- Automatic completion of missing valuation evidence.
- Any private financial data in the repository.

## Known Limitations

- This is alpha software.
- CLI behavior, file formats, schemas, report layouts, and extension contracts
  may change before `v1.0.0`.
- The CLI is the supported public alpha surface.
- `Reckonry.Api` is experimental and not part of the supported public alpha
  workflow.
- The Coinbase importer supports synthetic demo data and selected CSV fixtures
  only.
- Coinbase reconciliation validates synthetic aggregate metadata only.
- The Binance Italy workflow remains the only complete provider/country
  professional-review demo.
- Italy RW outputs are professional-review artifacts, not final filing forms.
- The Tax Dossier PDF is a review package, not a filing document.
- `reckonry explain` supports structured artifacts only; it does not explain PDF
  binaries.
- Plugin templates are source templates, not stable packages or runtime-loaded
  external binary plugins.
- Pricing providers are not installed in this release.
- No NuGet packages are published for this release.

## Privacy Notes

- The repository must not contain private exchange exports, private ledgers,
  taxpayer data, wallet addresses, transaction hashes, account identifiers, API
  keys, seed phrases, passwords, or generated reports based on private data.
- `input/`, `output/`, `artifacts/`, `.env`, `bin/`, `obj/`, and `.DS_Store`
  are ignored.
- Public samples under `samples/` are synthetic.
- Demo outputs under `artifacts/demo/` are local generated files and must not be
  committed.
- The demo does not require secrets, hosted services, telemetry, databases, or
  network calls after dependencies are restored.

## Professional Review Disclaimer

Reckonry does not determine tax liability, calculate taxes, provide filing
advice, or replace professional judgment.

Any tax-adjacent output must be reviewed by a qualified professional against
current official sources and complete taxpayer evidence before it is used for
accounting, audit, tax, legal, financial, or investment decisions.

Missing values remain missing. Reckonry must not invent financial data.
