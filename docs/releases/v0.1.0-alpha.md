# Reckonry v0.1.0-alpha

Release date: 2026-07-03  
Tag: `v0.1.0-alpha`

## What Reckonry Is

Reckonry is open-source infrastructure for reconstructing, validating, auditing, reconciling, and reviewing financial ledger evidence.

The project is not a tax calculator. It does not provide legal, tax, accounting, financial, or investment advice. It generates reproducible and explainable artifacts that can support professional review.

## What This Alpha Includes

- Canonical ledger v1 foundations.
- CLI-first local workflow.
- Binance normalized CSV importer demo workflow.
- Ledger validation.
- Audit, integrity, ledger, and summary reports.
- Binance Italy reconciliation module for comparing Reckonry outputs with provider-issued report inputs.
- Italy RW professional-review outputs:
  - RW snapshot report.
  - RW value report.
  - Accountant handoff package.
  - Accountant package in Markdown, CSV, and JSON.
- Tax Dossier PDF for professional review.
- Bundled plugin discovery for:
  - Importers.
  - Country tax modules.
  - Reports.
  - Reconciliation modules.
  - Pricing providers.
- Synthetic public demo data under `samples/demo/`.
- GitHub governance, issue templates, PR template, build/test/release workflows, and release documentation.
- Privacy posture for local-first processing, ignored private folders, fake public samples, and no telemetry.

## What This Alpha Does Not Include

- Tax filing.
- Tax certainty.
- Legal, tax, accounting, financial, or investment advice.
- A production API surface.
- Hosted SaaS workflows.
- Authentication or database persistence.
- NuGet package publishing.
- Stable external SDK guarantees.
- Stable plugin binary compatibility guarantees.
- Complete Binance product coverage.
- Real Coinbase, Kraken, Revolut, Crypto.com, or Bitstamp importers.
- Multi-country tax coverage beyond the Italy alpha module.
- Automatic market pricing.
- Automatic completion of missing valuation evidence.
- Any private financial data in the repository.

## Supported Demo Workflow

The supported public alpha workflow is the synthetic Binance Italy demo:

1. Import a fake Binance-style normalized CSV export.
2. Generate a canonical `ledger.json`.
3. Validate the ledger.
4. Generate audit and integrity reports.
5. Generate Italy RW snapshot and value reports.
6. Reconcile against fake Binance-style official report PDFs.
7. Generate an Italy RW accountant package.
8. Generate a Tax Dossier PDF for professional review.

The demo intentionally ends with `NOT READY FOR FILING`. That is expected. It proves that missing professional inputs are visible and are not invented.

## How To Run The Demo

Prerequisites:

- .NET SDK compatible with `net10.0`.
- macOS, Linux, or Windows PowerShell.

macOS/Linux:

```bash
dotnet build Reckonry.sln
dotnet test Reckonry.sln
scripts/demo.sh
```

Windows PowerShell:

```powershell
dotnet build Reckonry.sln
dotnet test Reckonry.sln
./scripts/demo.ps1
```

Generated demo outputs are written to ignored local files under `artifacts/demo/`.

## Known Limitations

- This is alpha software.
- CLI behavior, file formats, schemas, report layouts, and extension contracts may change before `v1.0.0`.
- The CLI is the supported public alpha surface.
- `Reckonry.Api` is experimental and not part of the supported public alpha workflow.
- Placeholder importer projects are discoverable but do not parse real provider files.
- The Binance importer covers the synthetic normalized demo workflow, not every Binance product or export format.
- Italy RW outputs are professional-review artifacts, not final filing forms.
- The Tax Dossier PDF is a review package, not a filing document.
- Demo output is synthetic and proves one provider/country workflow only.
- Pricing providers are not installed in this release.
- No NuGet packages are published for this release.

## Privacy Notes

- The repository must not contain private exchange exports, private ledgers, taxpayer data, wallet addresses, transaction hashes, account identifiers, API keys, seed phrases, passwords, or generated reports based on private data.
- `input/`, `output/`, `artifacts/`, `.env`, `bin/`, `obj/`, and `.DS_Store` are ignored.
- Public samples under `samples/` are synthetic.
- Demo outputs under `artifacts/demo/` are local generated files and must not be committed.
- The demo does not require secrets, hosted services, telemetry, databases, or network calls after dependencies are restored.

## Professional Review Disclaimer

Reckonry does not determine tax liability, calculate taxes, provide filing advice, or replace professional judgment.

Any tax-adjacent output must be reviewed by a qualified professional against current official sources and complete taxpayer evidence before it is used for accounting, audit, tax, legal, financial, or investment decisions.

Missing values remain missing. Reckonry must not invent financial data.
