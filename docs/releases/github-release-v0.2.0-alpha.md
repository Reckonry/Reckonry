# Reckonry v0.2.0-alpha

Release date: 2026-07-04

Reckonry `v0.2.0-alpha` is the second public alpha of Reckonry: CLI-first,
local-first infrastructure for reconstructing, validating, auditing,
reconciling, explaining, and reviewing digital asset ledger evidence.

This is alpha software. Reckonry is not a tax calculator, filing product,
accounting system, hosted service, or source of legal/tax certainty.

Canonical release notes:

- `docs/releases/v0.2.0-alpha.md`

## Highlights

- Coinbase CSV importer for synthetic demo data and selected CSV fixtures.
- Coinbase provider-level reconciliation against synthetic aggregate statement
  metadata.
- Expanded public demo with a Coinbase synthetic lane alongside the Binance
  Italy professional-review workflow.
- Alpha `dotnet new` plugin templates for importers, tax modules, reports, and
  reconciliation modules.
- `reckonry doctor` diagnostics for environment, plugin, privacy, SDK, demo, and
  repository checks.
- `reckonry explain` provenance output for supported structured artifacts.
- Snapshot/golden tests for public user-facing outputs.
- Updated README, architecture, importer SDK, reconciliation SDK, and template
  documentation.

## Quick Demo

```bash
dotnet build Reckonry.sln
dotnet test Reckonry.sln
scripts/demo.sh
```

Expected Italy alpha result: `NOT READY FOR FILING`.

That is intentional. The demo uses synthetic data and shows that missing
professional inputs are visible and are not invented.

## Notes

- Public demo data is synthetic.
- The CLI is the supported public alpha surface.
- `Reckonry.Api` is experimental and not part of the supported alpha workflow.
- Coinbase support is alpha-scoped and does not represent complete Coinbase
  product coverage.
- Release artifacts are generated by GitHub Actions after the tag is pushed.
