# Reckonry Design System

Reckonry should feel like mature engineering software: quiet, precise, auditable, and premium.

## Color Tokens

| Token | Hex | Usage |
| --- | --- | --- |
| `--lf-graphite-950` | `#0B0F14` | Primary dark surfaces, cover backgrounds |
| `--lf-graphite-900` | `#111821` | Elevated dark surfaces |
| `--lf-graphite-800` | `#1A222E` | Dark borders and dividers |
| `--lf-ink` | `#111827` | Main text on light surfaces |
| `--lf-muted` | `#536171` | Secondary text |
| `--lf-border` | `#D9E2EC` | Light borders |
| `--lf-paper` | `#F8FAFC` | Page background |
| `--lf-white` | `#FFFFFF` | Cards and text on dark |
| `--lf-forge-orange` | `#F97316` | Primary accent, verification mark |
| `--lf-electric-blue` | `#3B82F6` | Technical metadata and links |
| `--lf-success` | `#22C55E` | PASS status |
| `--lf-warning` | `#F59E0B` | WARNING status |
| `--lf-danger` | `#EF4444` | ERROR / NOT READY status |
| `--lf-neutral` | `#8B99A8` | NOT APPLICABLE status |

## Typography

Preferred typefaces:

- Inter: default UI and documentation typeface.
- Geist: excellent alternative for developer-product surfaces.
- IBM Plex Sans: useful for technical documents and data-heavy views.
- Manrope: good display alternative when a softer tone is needed.

Why: all four are modern, legible at small sizes, work well in dense technical interfaces, and avoid decorative personality that would weaken trust.

Fallback stack:

```css
font-family: Inter, Geist, "IBM Plex Sans", Manrope, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

## Spacing

Use an 8px spacing scale:

| Token | Value |
| --- | ---: |
| `--space-1` | `4px` |
| `--space-2` | `8px` |
| `--space-3` | `12px` |
| `--space-4` | `16px` |
| `--space-5` | `24px` |
| `--space-6` | `32px` |
| `--space-7` | `48px` |
| `--space-8` | `64px` |

## Corner Radius

- Small controls: `6px`
- Cards and panels: `10px`
- Hero/banner assets: `24px`
- App icon: use the provided asset radius only

Avoid oversized pill shapes except for compact status badges.

## Shadows

Use shadows sparingly.

- Light card shadow: `0 1px 2px rgba(11, 15, 20, 0.08)`
- Floating panel shadow: `0 16px 40px rgba(11, 15, 20, 0.16)`

Prefer borders and spacing over heavy shadows.

## Cards

Cards should be rectangular, compact, and information-dense.

- Background: `#FFFFFF`
- Border: `1px solid #D9E2EC`
- Radius: `10px`
- Padding: `16px` or `24px`

Do not nest cards inside cards.

## Buttons

Primary:

- Background: `#0B0F14`
- Text: `#FFFFFF`
- Accent: optional `#F97316` left border or icon

Secondary:

- Background: `#FFFFFF`
- Border: `#D9E2EC`
- Text: `#111827`

Danger:

- Background: `#EF4444`
- Text: `#FFFFFF`

## Status Colors

| Status | Color | Usage |
| --- | --- | --- |
| PASS | `#22C55E` | Valid, reconciled, complete |
| WARNING | `#F59E0B` | Needs professional review |
| ERROR / NOT READY | `#EF4444` | Blocks final output |
| NOT APPLICABLE | `#8B99A8` | Outside scope or unavailable |

Status labels must be explicit. Do not rely on color alone.

## Icons

Icons live in `assets/icons/`.

Style:

- Flat SVG.
- Rounded 6px strokes.
- No gradients.
- No clipart.
- Graphite base with orange/blue/green status accents.

## PDF Reports

PDFs should feel like product-grade review dossiers, not government forms.

Use:

- Dark premium cover section.
- White/light body pages.
- Compact cards for summary metrics.
- Clear status badges.
- Page footer with report title, page number, short ledger hash, and "Generated by Reckonry".

Do not use:

- Decorative financial imagery.
- Coin icons.
- Watermark patterns that reduce readability.
