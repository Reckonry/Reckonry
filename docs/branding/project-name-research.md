# Project Name Research

Research date: 2026-07-03

Current working name: Reckonry

Project position:

> Open-source digital asset ledger infrastructure for reconstruction, normalization, auditability, reconciliation, accounting workflows, and tax reporting infrastructure.

Brand philosophy:

> Build. Verify. Review.

This research treats the project as infrastructure, not a retail crypto tax calculator. The best name should work for a GitHub organization, NuGet packages, CLI, documentation, reports, and eventual company branding.

## Method

Availability checks were performed for the strongest candidates, not for every generated name.

- GitHub: checked `https://api.github.com/users/{slug}`, `https://api.github.com/orgs/{slug}`, `https://api.github.com/repos/{slug}/{slug}`, and, after API rate limiting, direct `https://github.com/{slug}` / `https://github.com/{slug}/{slug}` page status.
- Domains: checked registry RDAP where possible. `200` means registered/unavailable. `404` means likely available. `429` means rate-limited/uncertain.
- Domain sources: [IANA RDAP bootstrap](https://data.iana.org/rdap/dns.json), [Verisign .com RDAP](https://rdap.verisign.com/com/v1/), [Google Registry RDAP for .dev](https://pubapi.registry.google/rdap/), and [Identity Digital RDAP](https://rdap.identitydigital.services/rdap/).
- Trademark risk: high-level web search only. This is not legal advice and does not replace a trademark attorney search.

Known crowded naming space:

- Existing digital asset tax/accounting brands include [Ledgible](https://ledgible.io/), [Bitwave](https://www.bitwave.io/), [Cryptio](https://cryptio.co/), and [CoinLedger](https://coinledger.io/).
- Existing accounting/ledger software includes [Ledger CLI](https://ledger-cli.org/), [LedgerSMB](https://ledgersmb.org/), [SQL-Ledger](http://www.sql-ledger.com/), and [GnuCash](https://www.gnucash.org/).

## Scoring Scale

Scores are from 1 to 10 and are strategic naming scores before deep legal review.

- Memorability: how easily the name sticks.
- Pronunciation: how easy it is to say and spell aloud.
- Professionality: how suitable it feels for audit/accounting infrastructure.
- Developer appeal: how natural it feels in GitHub, packages, and CLI contexts.
- International appeal: how language-neutral it feels.
- Future scalability: whether it can grow beyond crypto tax.
- Uniqueness: how distinctive it appears before availability checks.

## Finalists

### 1. Verocord

Selected because it is short, pronounceable, professional, and directly evokes verified records without saying "crypto", "tax", or "ledger". It has the cleanest availability profile among the strongest names: GitHub handle/page not found, `{name}/{name}` repository not found, and `.com`, `.dev`, and `.io` all returned RDAP 404 at the time checked.

Best usage:

- GitHub organization: `verocord`
- CLI: `verocord`
- NuGet: `Verocord.Core`, `Verocord.Cli`, `Verocord.Reconciliation`, `Verocord.Tax.Italy`
- Report title: `Verocord Tax Dossier`

### 2. Provcord

Selected because it combines provenance and record, which matches the source-preservation philosophy. Availability is also excellent. The drawback is that it is slightly harder to say and less polished than Verocord.

### 3. Reconto

Selected because it suggests reconstruction, reconciliation, and accounting without being too literal. It is highly memorable and international. The drawback is that `reconto.com` is registered, so the brand may need `reconto.dev` or `reconto.io`.

### 4. Provixa

Selected because it has a modern infrastructure feel and suggests provenance/verification. GitHub and `.dev`/`.io` availability look good. The drawback is that `.com` is registered and the meaning is less obvious than Verocord or Reconto.

### 5. Accruva

Selected because it feels natural in accounting workflows and can scale to enterprise finance. It is less crypto-coded than Reckonry. The drawback is that it leans closer to accounting than immutable ledger reconstruction and `.com` is registered.

### 6. Validara

Selected because it communicates validation and professional review. It is polished and extensible. The drawback is that "valid" names are crowded and the `.com` is registered.

### 7. Veratrail

Selected because it cleanly expresses audit trails and explainability. It is more descriptive than Verocord but still brandable. The drawback is that `.com` is registered.

### 8. Auditline

Selected because it is immediately understandable for accountants and auditors. It works for reports and documentation. The drawback is that it is less unique and more service-like than platform-like.

### 9. Statora

Selected because it is short and professional, with a subtle state/record feeling. The drawback is weaker semantic clarity and `.com`/`.io` are registered.

### 10. Normaledger

Selected because availability is excellent and the meaning is direct: normalized ledger. The drawback is that it is clunkier and less premium than the top coined names.

## Recommended Winner

## Verocord

Recommended brand line:

> Verocord  
> Build. Verify. Review.

Alternative descriptive line:

> Verocord  
> Open digital asset ledger infrastructure.

Why Verocord is superior to Reckonry:

- It is more ownable. "Ledger" and "Forge" are both crowded software words; Verocord is coined but still meaningful.
- It better expresses the core value. The product is not forging ledgers; it is verifying records from preserved source evidence.
- It avoids crypto cliches. It does not sound like an exchange, token, wallet, meme project, or retail tax calculator.
- It scales better. Verocord can cover importers, reconciliation, evidence preservation, audit reports, tax modules, accounting integrations, and future compliance workflows.
- It works cleanly in developer contexts: `verocord`, `Verocord.Core`, `Verocord.Cli`, `Verocord.ReportSdk`, `Verocord.Reconciliation`.
- It has the best combined availability signal among the high-scoring names: GitHub handle/page not found, GitHub `{name}/{name}` repo not found, and `.com`, `.dev`, and `.io` all appeared available by RDAP.

Recommended next steps before renaming:

1. Run a formal trademark search in the US, EU, UK, and relevant operating markets.
2. Register `verocord.com`, `verocord.dev`, and `verocord.io` immediately if still available.
3. Reserve the GitHub organization `verocord`.
4. Reserve NuGet package prefixes before public announcement.
5. Keep `Reckonry` as an internal migration codename until all package, docs, and repo moves are planned.
